FALO Prompt Manager Design

Capability Pack 藍圖

Capability Pack 保留為 FALO Runtime 階段藍圖；FALO Prompt Manager v0.3 先專注 Prompt Asset Management。

1. 核心句

先把 Prompt 做成產品；Capability Pack 留在藍圖。

殼公開、卡分級、離線先匯入、線上接 GAS 是 Runtime 階段方向，不是 v0.3 主線。

2. 三層架構

Public PWA Shell
  ↓
Offline Capability Pack JSON
  ↓
Future GAS Online Capability Pack

這是未來 Runtime 架構：Shell 顯示功能槽位與鎖定狀態；離線 pack 啟用部分功能；GAS 線上 pack 回傳授權能力。v0.3 不實作。

3. 功能槽位

SlotCapability未啟用時顯示
Prompt 管理prompt.basic需要 Basic Pack 或 FALO GAS 啟用
OCR 功能卡槽ocr.postprocess需要 OCR Pack 或 FALO GAS 啟用
Voice 功能卡槽voice.dictation需要 Voice Pack 或 FALO GAS 啟用
JSON Connectjson.connect需要 Basic Pack 或 FALO GAS 啟用
Workflow / RFPworkflow.rfp需要 RFP Pack 或 FALO GAS 啟用
GAS 功能包gas.connect預留，尚未連線

5. 最小 UI 改版

  • Topbar 增加「匯入功能卡包」與 Pack 數量。
  • Sidebar 增加「功能卡槽」列表。
  • Main 未啟用功能顯示鎖定訊息。
  • Inspector 顯示已啟用 Pack 與 GAS 預留欄位。

6. GAS 預留

PWA Shell
  ↓
輸入 GAS Endpoint / Account / Token
  ↓
FALO GAS Web App
  ↓
依帳號回傳 capability packs
  ↓
PWA Shell 啟用對應功能

這是 FALO Runtime 階段的設計，不是 FALO Prompt Manager v0.3 的實作項目。目前不做登入、不送出帳密、不儲存 token。

7. 目前下一步

目前不建議為 Capability Pack 動 index.html。近期若要改介面，應優先強化 Prompt Card 欄位、Workflow Step、搜尋、標籤、狀態、變數代入與一鍵複製。
