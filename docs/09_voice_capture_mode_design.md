# Voice Capture Mode Design

本文用途：記錄 FALO Prompt Manager v1.01 後續版本中的語音輸入設計觀念，方便 AI、協作者與後續維護者理解。

---

## 1. 核心觀念

語音插入不是一顆單純按鈕，而是一個短暫的輸入模式。

使用者按下語音後，系統應該清楚回答三件事：

1. 現在有沒有在聽？
2. 語音文字會寫到哪裡？
3. 說完要怎麼停止？

一句話：

> 語音插入是一種捕捉模式；啟動時，按鈕、編輯區與右側 Voice 卡都要一起說明目前狀態。

---

## 2. 兩種語音模式

### 2.1 全域 Voice_Text

全域模式用來把口述內容寫入 `[Voice_Text]`。

適合：

- 口述 idea
- 課堂即席補充
- 會議片段
- 現場訪談摘要

資料流：

```text
開始聽寫
↓
Web Speech API
↓
[Voice_Text]
↓
Prompt Card 套用變數
```

### 2.2 編輯區語音插入

編輯模式用來把口述內容插入目前游標所在文字框。

適合：

- 直接口述 Prompt 內容
- 補充用途說明
- 口述預期輸出
- 口述人工檢查點

資料流：

```text
點選文字框
↓
按「語音插入」
↓
Web Speech API
↓
插入目前游標位置
↓
即時預覽更新
```

---

## 3. UI 狀態設計

### 3.1 按鈕狀態

平常：

```text
語音插入
```

聽寫中：

```text
聽寫中
```

視覺上使用藍紫色亮起，代表麥克風正在捕捉。

### 3.2 右側 Voice 卡

右側 Voice 卡不只是輸入框，而是語音捕捉控制台。

它應顯示：

| 欄位 | 說明 |
| --- | --- |
| 目前模式 | 全域 Voice_Text / 編輯區語音插入 |
| 寫入位置 | `[Voice_Text]` 或目前文字欄位名稱 |
| 使用方式 | 先點文字框、按語音插入、說話、停止語音 |

### 3.3 編輯區提示

當使用者點到某個文字框，系統可以預先顯示：

```text
語音會插入：Prompt 內容編輯
```

按下語音插入後：

```text
正在聽寫，文字會插入「Prompt 內容編輯」。說完請按「停止語音」結束。
```

---

## 4. 教學說法

可以這樣和初學者說：

> Voice 是口述食材。你可以把它放進 `[Voice_Text]`，也可以直接把它倒進正在編輯的 Prompt 欄位。

更口語一點：

> 先點你要寫字的地方，再按語音插入。系統會告訴你目前正在聽、會寫到哪裡。

---

## 5. 本版不做

- 不做完整語音平台。
- 不做音檔上傳。
- 不做錄音保存。
- 不做語者分離。
- 不做雲端語音 API。

本版只使用瀏覽器 Web Speech API，作為最輕量的語音輸入入口。
