FALO Prompt Manager

產品策略對齊

把 v0.3 收斂回 FALO Prompt：先把 Prompt Asset Management 做到好用、可教、可展示；Capability Pack 保留在 Runtime 藍圖。

1. 第一原則

先把 Prompt 做成產品。Capability Pack 留在藍圖。不要急著把 Runtime 搬進來。

目前 FALO Prompt Manager 還在產品雛型階段，不需要太早處理授權、Marketplace、金流、完整 GAS 派送或 Runtime 功能治理。

2. 三條產品線

FALO Book

學習、閱讀、教材。

核心：Knowledge。

目的：讓使用者理解。

FALO Prompt

Prompt Asset Management。

核心：Skill。

目的：快速套用與共創。

FALO Runtime

能力執行平台。

核心：Execution。

目的：讓事情真正發生。

3. 本專案位置

FALO Prompt Manager
  = FALO Prompt
  = Prompt Asset Layer
  ≠ FALO Runtime

Capability Pack 可以留在設計文件、schema 與 roadmap，但不要讓它成為 v0.3 主功能。
