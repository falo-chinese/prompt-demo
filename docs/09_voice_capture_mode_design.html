FALO Prompt Manager · Voice

Voice Capture Mode Design

語音插入不是一顆單純按鈕，而是一個短暫的輸入模式。啟動時，按鈕、編輯區與右側 Voice 卡都要一起說明目前狀態。

核心 模式 UI 狀態 教學說法 邊界

1. 核心觀念

使用者按下語音後，系統要清楚回答三件事：現在有沒有在聽、語音文字會寫到哪裡、說完要怎麼停止。

語音插入是一種捕捉模式；啟動時，按鈕、編輯區與右側 Voice 卡都要一起說明目前狀態。

2. 兩種語音模式

3. UI 狀態設計

位置設計
按鈕平常顯示「語音插入」，聽寫中顯示「聽寫中」並以藍紫色亮起。
右側 Voice 卡顯示目前模式、寫入位置與使用方式。
編輯區提示提示語音會插入目前文字框，說完按停止語音。

4. 教學說法

Voice 是口述食材。你可以把它放進 [Voice_Text]，也可以直接把它倒進正在編輯的 Prompt 欄位。

先點你要寫字的地方，再按語音插入。系統會告訴你目前正在聽、會寫到哪裡。

5. 本版不做

  • 不做完整語音平台。
  • 不做音檔上傳。
  • 不做錄音保存。
  • 不做語者分離。
  • 不做雲端語音 API。

本版只使用瀏覽器 Web Speech API，作為最輕量的語音輸入入口。
